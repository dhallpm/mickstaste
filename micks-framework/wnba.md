# Micks Picks WNBA Framework

## Card Standard
- WNBA cards require confirmed odds, current injury/news timing, and a clear matchup reason.
- Smaller rotation pools make minutes security and role changes especially important.
- Default to review mode when lineup or usage information is incomplete.

## Core Scoring
- Calculate implied probability from the offered price.
- Compare market price to estimated true probability before assigning a grade.
- EV edge must be shown in notes when enough provider data exists.
- Best Number and No Bet Cutoff must be price-specific because WNBA markets can move quickly.

## Grading
- A = strong edge with secure minutes, clear matchup advantage, current injury context, and a playable number.
- B = positive edge with manageable volatility or one unresolved but non-critical factor.
- C = small edge that is playable only at the posted number and unit size.
- Pass = stale price, uncertain availability, thin data, low-liquidity move, or no true probability edge.

## Matchup Layer
- Evaluate pace, offensive efficiency, defensive scheme, and rebounding profile.
- Usage concentration matters: a single injury can materially shift shot volume, assist equity, and defensive attention.
- Fatigue and travel matter more when a team has a short bench or repeated starter load.
- Regression spots include extreme three-point variance, free-throw rate spikes, and opponent shooting outliers.

## Props Logic
- Prioritize minutes security, usage rate, matchup allowance, and line sensitivity.
- Avoid props when player role is uncertain or the book moved through the best number.
- Player props must route to Props Lab only when they are true player props.
- Team totals, moneylines, spreads, and game totals are not Props Lab content.

## Market Discipline
- Treat low-liquidity market movement carefully. A move can signal information or simply expose a thin board.
- Use no-bet cutoffs aggressively on props and smaller-market sides.
- Public-team inflation applies when star narratives or recent visible games distort price.

## Safe Version Vs Value Version
- Safe version: the cleanest expression with the strongest role and matchup certainty.
- Value version: plus-money or alt-line expression only when the provider data supports the extra variance.
- If safe and value versions conflict, publish the safer card and keep the value version in notes or review.
